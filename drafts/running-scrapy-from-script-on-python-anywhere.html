Running Scrapy From a Script on Python Anywhere (Scrapy Part 2)

Posted by Aly Sivji in Tutorials   

(Note: This post is part of my reddit-scraper series)

Summary

  • Overview of MongoDB
  • Discussion of Object-Relational Mapping (ORM)
  • Use MongoEngine to get items out of MongoDB
  • Render HTML pages using the Jinja2
  • Interact with REST API to send emails with Requests

Previously on Siv Scripts, we implemented a Scrapy pipeline to store Top Posts scraped from Reddit into a MongoDB collection. Right now we have to manually our spider once a week so let's explore how to make this process a script and have it run on Python Anywhere

http://scrapy.readthedocs.io/en/latest/topics/practices.html#run-from-script

Note on Unit Tests: * jury is out for scrapers * unit test with fake responses (http://stackoverflow.com/questions/6456304/scrapy-unit-testing http://stackoverflow.com/questions/27977050/how-can-i-start-to-write-unit-test-in-web-scrapy-using-python) * Contracts are good (http://stackoverflow.com/questions/6456304/scrapy-unit-testing/38214137#38214137), contracts might be depreciated (http://stackoverflow.com/questions/38191402/scrapy-contracts-with-multiple-parse-methods)

Reddit is pretty static so i really don't want to go down the unit test route. if the script breaks, i'll notice via the emails that are sent right after. For projects involving websites that change a lot more than reddit, i would set up some unit tests. for our purposes, email is a good enough check.
