Ultimate Solution to Python Virtual Environments: pyenv + virtualenvwrapper

Posted by Aly Sivji in Drafts   

Playing around on the command line is one of my favourite things about programming. It wasn't always this way. I used to worry that the commands I entered would destory the delicate balance between everything working and a reformat. Would this be the pip install that ended it all?

Enter virtual environments.

A virtual environment (venv) is a tool that allows us to keep project dependencies isolated. If Project A requires pandas version 0.17 and Project B requires pandas version 0.21, we can create two virtual environments to separate each project and keep workflows independent. More info in the Python documentation.

It's been around a year since I started using venvs for each of my projects, but I had yet to find the perfect development workflow.

Tried Anaconda; it was too bloated for non-Data Science work.

Next came virtualenv + virtualevnwrapper. Did everything I wanted, but had to manually manage Python installations when using different versions of Python.

Like everybody who learns Docker, I went thru a containerize all the things phase. The non-existent battery life on my Mac told me this wasn't the most feasible solution.

Then I found pyenv.

In this post, I will describe how to install pyenv and pyenv-virtualenvwrapper. I will also walk through the steps required to setup and use a new virtual environment. This is mostly to document the steps for my benefit, but hopefully it will help others in the same boat.
