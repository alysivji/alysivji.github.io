Bashmarks: Navigate The Command Line With Ease

Posted by Aly Sivji in Drafts   

Even though my development folders are well organized, I still have to feel my way around the filesystem when I'm looking for a project directory. It's like trying to find an exit in a dark room: a cd here, an ls there, maybe a find when I'm stuck.

I usually find what I'm looking for, but not without some frustration along the way. Plus, all the seconds it takes to navigate to the right project directory start to add up after a while.

Then I discovered Bashmarks. It's an absolute game changer, I can navigate the filesystem with ease. Makes me feel a bit like Dr. Who.

In this post we will explore Bashmarks, install it, and walk through some common commands.

Bashmarks

Bashmarks is a [bash] shell script that allows you to save and jump to commonly used directories

Use another shell? Not a problem. A quick Google helped me find ports for fish and zsh shells.

Installation

Download the files into a temporary directory and install using GNU make

mkdir temp
git clone git://github.com/huyng/bashmarks.git
cd bashmarks
make install

Add the following line to ~/.bash_profile:

source ~/.local/bin/bashmarks.sh

Reload the shell:

source ~/.bash_profile

And we're good to go.

Commands

Bashmarks has a very simple interface:

We can save bookmarks using the s [bookmark_name] command:

Go to directory, pwd, s <bookmark_name> to save

List using l

Then g, pwd, and g to bookmark we created. and pwd

s <bookmark_name> - Saves the current directory as "bookmark_name"
g <bookmark_name> - Goes (cd) to the directory associated with "bookmark_name"
p <bookmark_name> - Prints the directory associated with "bookmark_name"
d <bookmark_name> - Deletes the bookmark
l                 - Lists all available bookmarks

Have gifs showing each of these.

Tips

  • Bashmark shell commands
  • Have a current project folder
  • It supports tab completion (show gif, type curr and tab to go to current project)

Conclusion

Bashmarks is a great tool that increases terminal productivity. If you have you additional terminal tips and/or tricks, please share them in the comments below.
