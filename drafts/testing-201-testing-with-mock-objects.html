Other Types of Tests

There is a lot more to testing than just the Test Pyramid. Other types of testing include

  • manual test - test completed by human
  • load/scale test - impose stress on application by simulating heavy load conditions / highly concurrent load conditions
  • infrastructure test - testing infrastructure underlying software
  • UAT (user acceptance testing) - human test assess implementation of software to see if it meets specifications
  • acceptance test (http://softwaretestingfundamentals.com/acceptance-testing/)
  • smoke test, just try something to see if it works. will give off smoke if it does not

Testing Anti-Patterns

  • tests should be focused
  • test one thing at a time, if we have multiple things that we are testing for. a test failure won't provide any additional information. we're gonna have to do the work to dig into it, so save yourself a headache and make your tests indicate where to look. this is more true for unit tests
  • too many test cases that are the same
  • testing happy path vs testing bad path. this is because we aren't taught about how to test properly. writing tests feels like a chore. if we have the right tools, it won't. this series of posts is trying to fix that.
